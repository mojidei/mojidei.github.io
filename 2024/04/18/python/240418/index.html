初探部署

先复习一下敏捷

举例来说，SpaceX 公司想造一个大推力火箭，将人类送到火星。但是，它不是一开始就造大火箭，而是先造一个最简陋的小火箭 Falcon 1。结果，第一次发射就爆炸了，直到第四次发射，才成功进入轨道。然后，开发了中型火箭 Falcon 9，九年中发射了70次。最后，才开发 Falcon 重型火箭。如果 SpaceX 不采用迭代开发，它可能直到现在还无法上天。

既然我这个“小火箭”都能回应请求了，那么也该“送上天”了。

关键点在哪里？

python相对于java来说有点不一样……

WSGI (Python Web Server Gateway Interface)指定了web服务器和Python web应用或web框架之间的标准接口。

根据文档，部署需要创建一个分发文件(.whl)，设置新的虚拟化环境，init-db，创建 config.py 文件并配置密钥。生产环境要使用（另）一个WSGI组件提供服务器功能，不能用内置的开发服务器（flask run）。

WSGI中比较有名的有gunicorn，logo是一个绿色的独角兽.

还提到要搞一个py的虚拟环境……否则部署多个项目到一个机器会乱套。但是都上虚拟环境了那么就来玩docker吧！

用docker部署flask的教程既有用flask内置的开发服务器也有用gunicorn的。
